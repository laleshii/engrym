# Authoring commands

`src/commands/author.rs` implements `new` (create), `set` (update), and `rm`
(delete). They operate on the Markdown files — the source of truth — not the
index, and locate documents by scanning for their frontmatter `id`, so they
work even when the index is stale.

## Why generate frontmatter

Frontmatter is generated by serializing the [[frontmatter-schema]] struct, never
hand-written, so an agent supplying fields as flags always gets a
guaranteed-valid, reviewable document. `rm` warns about inbound references that
would dangle. After authoring, rebuild with `engrym index`.
